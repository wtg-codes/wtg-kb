# WTG-KB Response: Improvement & Refinement Plan

This document outlines the engineering response to the Level 10 Critique. Each issue will be addressed with a specific, actionable plan to move the project from a "skeleton" to a "production-grade brain."

---

## 1. Design & Architecture Refinement

### 1.1 Consolidation of Rules
- **Action**: Delete `.julesrules` and create a symbolic link from `.julesrules` to `.cursorrules`, OR update the CI/CD to automatically sync them. Ideally, move shared logic to a root `AGENT_GUIDE.md` and have both rule files reference it to minimize duplication.
- **Sidebars**: Transition `sidebars.js` to use the `autogenerated` category type. This ensures that new documentation files are instantly visible without manual overhead.

### 1.2 Documentation Content Strategy
- **"Content Injection Phase"**: Prioritize the expansion of placeholder files.
  - **Vertex AI**: Populate with actual WIF-enabled Python snippets and Cloud Run deployment patterns.
  - **Architecture**: Add Mermaid.js templates for common GCP patterns (3-tier app, event-driven data pipeline).
- **Submodule Management**: Implement a "Knowledge Indexer" script that scans the `imported/` directory and generates a `KNOWLEDGE_MAP.md` to make external rules searchable by the agent.

---

## 2. Automation & Scripting Overhaul

### 2.1 Robust Meta-Syncing
- **JSON-Driven Config**: Instead of using `sed` on Javascript files, move project metadata (ORG, REPO, URLs) into a `project.json` file.
- **Update Scripts**: Rewrite `scripts/sync-project-meta.sh` to read from `project.json` and use a proper template engine or a more robust parsing logic to update Docusaurus and README files.

### 2.2 Shell Scripting Standards
- **Global Safety**: Apply `set -euo pipefail` to all scripts in `scripts/`.
- **Structural Validation**: Implement `markdown-lint` or a custom Python-based validator for link integrity. Move structural checks out of the full Docusaurus build process in `test-kb.sh` to speed up the feedback loop.

### 2.3 Functional Integration
- **MCP Manager v2**: Enhance `scripts/mcp-manage.sh` to actually interact with `package.json` to "install" or "link" MCP servers from the submodules into a local dev environment.
- **Trend Scraper**: Update `fetch-trends.sh` to use a list of URLs and implement basic error handling/fallback logic.

---

## 3. Performance & DX Enhancements

### 3.1 Design System
- **CSS Variables**: Refactor `custom.css` to use a more structured set of CSS variables inspired by Tailwind's color palette.
- **Componentization**: Move glassy UI elements into reusable Docusaurus components (MDX) to ensure consistency across the site.

### 3.2 Advanced Compliance
- **Link Checker**: Integrate a proper link-checking tool (like `lychee`) into the Compliance Bot to verify both local and remote URLs.
- **PR Automation**: Set up a GitHub Action that runs the Compliance Bot on every PR and comments with specific fix suggestions.

---

## Implementation Roadmap
1. **Sprint 1 (Robustness)**: Shell script safety, project.json implementation, and rule consolidation.
2. **Sprint 2 (Content)**: Deep-dive into Vertex AI, Architecture, and GWS documentation.
3. **Sprint 3 (Tools)**: MCP Manager v2 and Advanced Compliance Bot.
