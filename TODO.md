# WTG Knowledge Base Roadmap: From Skeleton to Brain

This roadmap outlines the prioritized steps to evolve the WTG-KB into a production-grade Solution Architect Knowledge Base.

## Phase 1: Structural Integrity & Industrialization (High Priority)
*Objective: Ensure the platform is robust, safe, and easily maintainable.*

- [x] **Script Safety**: Implement `set -euo pipefail` across all shell scripts.
- [x] **Rule Consolidation**: Synchronize `.cursorrules` and `.julesrules` via symlinking.
- [x] **Data-Driven Config**: Move project metadata to `project.json` and refactor sync scripts.
- [x] **Dynamic Navigation**: Automate sidebars in Docusaurus using `autogenerated` categories and dynamic `llms.txt`.
- [x] **Advanced Compliance v2**:
    - [x] Integrate `lychee` for external link validation in CI/CD.
    - [x] Implement `markdownlint` to enforce documentation quality.
    - [x] Add a "stale content" detector for documentation not updated in > 90 days.

## Phase 2: Content Injection & Expert Knowledge (Crucial)
*Objective: Replace placeholders with high-value, expert-level patterns and snippets.*

- [ ] **Cloud & Architecture Deep-Dive**:
    - **Vertex AI**: Add production patterns for Vertex AI Pipelines and WIF-based authentication.
    - **Cloud Run**: Document blue/green deployment strategies and sidecar patterns.
    - **Mermaid Patterns**: Populate `docs/architecture/patterns.md` with diagrams for 3-tier apps, event-driven architecture, and multi-region failover.
- [ ] **Workspace & Apps Script**:
    - Document `gas-fakes` patterns for CI/CD testing of Apps Script.
    - Add "Performance Gold Standards" for large-scale Google Sheets processing (inspired by Kanshi Tanaike).
- [ ] **Security First**:
    - Complete the `wif-patterns.md` with concrete Terraform/gcloud commands.
    - Document Chrome Enterprise Premium (CEP) data ingestion patterns.
- [ ] **Linting & Quality**: Fix initial batch of markdownlint warnings across the codebase.

## Phase 3: Agentic Tooling & Integration (Advanced)
*Objective: Enhance the KB's utility for AI Agents and automated workflows.*

- [ ] **MCP Ecosystem**:
    - Upgrade `scripts/mcp-manage.sh` to support dynamic installation and linking of MCP servers from submodules.
    - Create a catalog of "Recommended MCP Servers" for Solution Architects.
- [ ] **RAG Optimization**:
    - Implement a "Knowledge Indexer" to scan `imported/` submodules and generate a metadata-rich `KNOWLEDGE_MAP.md`.
    - Enhance `vertex-ai-sync.sh` to support metadata injection for better Vertex AI Search indexing.
- [ ] **Interactive Sandbox**:
    - Add a live Mermaid editor or a "Pattern Prototyper" using Docusaurus MDX components.

## Phase 4: Lifecycle & Curation (Maintenance)
*Objective: Keep the KB fresh and relevant.*

- [ ] **Automated Curation Inbox**: Scripted "scouts" to monitor GDE blogs and GitHub for new trends.
- [ ] **PR Automation**: GitHub Action to auto-verify and provide feedback on documentation quality for every PR.
